#### El `main.c` reorganizado

La diferencia entre el `main.c` del 28 y el del 29 es notable:

**Versión 28 (inicial):**
```c
Opa_ref_Start();
VDAC_Start();
Opa_buff_1_Start();
Opa_buff_2_Start();
Opa_buff_3_Start();
```

**Versión 29 (reorganizada):**
```c
LPF_ref_Start();    // LPF antes del Opa_ref — cambia el orden
LPF_ADC_Start();    // nuevo: LPF específico para el ADC
Opa_ref_Start();
PGA_ref_Start();    // nuevo: PGA de referencia separado del PGA de señal
VDAC_Start();
ADC_Start();
PGA_Start();        // renombrado de PGA_1 a PGA
```

Los cambios revelan una reorganización del esquema analógico:

1. **Tres buffers eliminados (`Opa_buff_1/2/3`)**: Los LSTs del 29 muestran `Opa_buff_1_PM.lst` y `Opa_buff_2_PM.lst` (Power Management), lo que indica que los buffers se cambiaron a un modo de bajo consumo o que se rediseñó el routing analógico. El bloque `Opa_1V024` aparece como nuevo — un op-amp configurado específicamente para la referencia de 1.024V (voltaje de referencia del ADC del PSoC).

2. **`LPF_ref` y `LPF_ADC` separados**: En lugar de dos LPFs genéricos (`LPF_1`, `LPF_2`), ahora hay dos con roles explícitos:
   - `LPF_ref`: filtra la señal de referencia de voltaje (eliminar ruido en la referencia del ADC)
   - `LPF_ADC`: filtra la señal antes del ADC (anti-aliasing del martillo)

3. **`PGA_ref` separado de `PGA`**: El PGA de referencia ajusta el nivel de offset/bias, mientras que el PGA principal ajusta la ganancia de la señal del martillo.

#### `Opa_1V024` — la referencia interna de 1.024V

El bloque `Opa_1V024` en los archivos de debug indica que se está usando el op-amp interno del PSoC como seguidor de tensión (voltage follower) para la referencia de 1.024V del chip. Esta referencia es generada internamente por el PSoC y es muy estable (±1%), lo que la hace ideal como referencia del ADC.

La arquitectura resultante de la cadena del martillo:

```
Martillo PCB Piezotronics 086D20
    │ (señal ±5V proporcional a fuerza)
    │
    ▼
[PGA] — ganancia ajustable (1× a 32×), adapta la señal al rango del ADC
    │
    ├── [LPF_ADC] — filtro anti-aliasing para el ADC (fc configurable)
    │        │
    │        ▼
    │      [ADC] — SAR ADC (alta velocidad, baja latencia)
    │
    └── [LPF_ref] ─── [Opa_1V024] ─── [PGA_ref] ─── [VDAC]
         (filtrado de la referencia → estabilidad del offset)
```

Esta cadena tiene sentido para el martillo: necesitas alta velocidad de muestreo (para capturar el pulso de 1-10ms con buena resolución temporal), ganancia ajustable (el golpe puede ser suave o fuerte), y un filtro anti-aliasing claro (no quieres que el rebote del martillo a 200-300ms después aliase en la banda de interés).
