### Cambio 3 — deiStation: Tailscale SSH

La línea añadida al `.zsh_tesis`:

```zsh
alias deiStationUp="sudo tailscale up && ssh ealvarez@100.108.177.6"
```

Es la primera mención de un servidor o máquina remota en el proyecto. Tres elementos:

**`sudo tailscale up`** — Activa Tailscale, una VPN mesh peer-to-peer. Tailscale crea una red privada entre dispositivos del mismo usuario sin necesidad de una VPN centralizada. La IP `100.108.177.6` es una dirección Tailscale (rango 100.x.x.x).

**`ssh ealvarez@100.108.177.6`** — Conexión SSH a una máquina con el usuario `ealvarez`. Dado que el nombre es `deiStationUp` ("deiStation" = nombre de la máquina, posiblemente del usuario "dei" o "deiStation"), se puede inferir que es una PC del laboratorio, un servidor de la universidad, o una Raspberry Pi / PC dedicada al proyecto.

**Implicaciones para el proyecto:** El usuario está configurando un ambiente de trabajo distribuido. Puede haber:
1. Una máquina de escritorio potente (para MATLAB, por ejemplo) accesible remotamente
2. Una Raspberry Pi o similar conectada al hardware del geófono
3. Una PC del laboratorio de geofísica

La presencia de este alias sugiere que el trabajo experimental comienza a involucrar más de un dispositivo. El hardware MASW que se va a construir tiene componentes que corren en diferentes plataformas: PSoC (firmware C), ESP32 (firmware C++), y PC de procesamiento (MATLAB).
