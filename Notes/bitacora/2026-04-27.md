### Desarrollo detallado: `geophone_scope_simple.m` — El scope reducido

#### Por qué se necesitaba una versión simplificada

El `geophone_scope.m` del commit `c3d8ceb1` (22 de abril) tenía 657 líneas y manejaba 4 canales con un protocolo de 10 bytes:

```
[0xAA][0x00][sar_hi][sar_lo][ds2][ds1][ds0][flt2][flt1][flt0]
```

Cuatro canales: SAR 12-bit, DelSig 18-bit, DFB filter, y un canal digital MATLAB.

Pero el proyecto `ComparacionDiferencial.cydsn` es una placa diferente con un firmware diferente. Ese PSoC no tiene el ADC_SAR del circuito original, solo DelSig + DFB (filtro digital en hardware). El protocolo entonces cambia a **8 bytes**:

```
[0xAA][0x00][ds2][ds1][ds0][flt2][flt1][flt0]
```

Crear una versión simplificada del scope era necesario para poder trabajar con este nuevo circuito analógico diferencial sin tener que mantener compatibilidad con el circuito original. Es un movimiento inteligente: en lugar de hacer el scope original más complejo, se bifurca en una versión especializada.

#### Estructura de `geophone_scope_simple.m`

El script mantiene la misma arquitectura que el original (struct de estado `S`, timer a 20ms, parser por estados), pero con todo lo que sobra eliminado:

```matlab
function geophone_scope_simple()
% geophone_scope_simple — Interfaz PSoC5 entrada real (DelSig + DFB)
%
% Protocolo RX (PSoC -> PC, 8 bytes, USBUART CDC):
%   [0xAA][0x00][ds2][ds1][ds0][flt2][flt1][flt0]
%   ds  = int24 big-endian signed  (ADC_DelSig 18-bit, ±6.144V)
%   flt = int24 big-endian signed  (salida DFB)

    S.fs    = 1500;
    S.ds_mV = 1000.0 / 21333.0;   % 1 LSB ≈ 0.04688 mV
```

`1000.0 / 21333.0` es la conversión del ADC_DelSig en configuración CFG1 (±6.144V, 18 bits):

```
Rango = 2 × 6.144 V = 12.288 V
Counts = 2^18 = 262144
LSB = 12288 / 262144 ≈ 0.04688 mV = 1000/21333 mV
```

Esta es la misma escala que en la versión grande.

#### Parser de 8 bytes

El parser es una máquina de estados de 2 estados: HUNT (estado 0) y COLLECT (estado 1):

```matlab
if S.parseState == 0
    if b == uint8(0xAA)
        S.pktBuf(1) = b;
        S.pktIdx    = 1;
        S.parseState = 1;
    end
    i = i + 1;
else
    S.pktIdx           = S.pktIdx + 1;
    S.pktBuf(S.pktIdx) = b;
    i = i + 1;

    if S.pktIdx == 8
        pkt = double(S.pktBuf);

        % Byte 2 = tipo: si es 0x01 es HEARTBEAT
        if pkt(2) == 1
            logMsg(sprintf('HEARTBEAT  total_bytes=%d', S.totalBytes));
            S.parseState = 0;
            continue;
        end

        % Bytes 3-5: DelSig int24 big-endian
        ds_u = pkt(3)*65536 + pkt(4)*256 + pkt(5);
        if pkt(3) >= 128, ds_v = ds_u - 16777216; else, ds_v = ds_u; end

        % Bytes 6-8: DFB filter int24 big-endian
        flt_u = pkt(6)*65536 + pkt(7)*256 + pkt(8);
        if pkt(6) >= 128, flt_v = flt_u - 16777216; else, flt_v = flt_u; end

        newDs (end+1, 1) = ds_v;
        newFlt(end+1, 1) = flt_v;

        S.parseState = 0;
    end
end
```

La decodificación del int24 big-endian es la misma técnica que en el scope original:
- Reconstruir el entero sin signo: `u = b2×65536 + b1×256 + b0`
- Si el MSB está seteado (≥128 → bit de signo en int24): `v = u - 16777216` (16777216 = 2²⁴)

La detección de HEARTBEAT en `pkt(2) == 1` sugiere que el firmware de `ComparacionDiferencial` tiene un mecanismo de keep-alive (un paquete especial `[0xAA][0x01]...` que el PC puede usar para verificar que el PSoC sigue vivo).

### Desarrollo detallado: `Compensador.m` — Análisis VCVS como compensador activo

Este archivo es el más conceptualmente significativo del commit. Mientras que `compensadorKai.m` y `compensadorPazos.m` implementan compensación **digital en MATLAB**, `Compensador.m` explora compensación **analógica activa** usando un filtro VCVS (Voltage Controlled Voltage Source).

#### ¿Qué es un filtro VCVS?

Un filtro VCVS (también llamado filtro Sallen-Key) es una topología de filtro activo de segundo orden que usa un op-amp en configuración de seguidor de tensión controlado. La ganancia de la realimentación determina el factor de calidad Q del filtro.

Para compensar el geófono analógicamente, la idea es poner un circuito activo en la salida del geófono que "corrija" la respuesta en frecuencia antes de digitalizar.

#### Parámetros del circuito

```matlab
R1 = 150e3; Ra = 10e3; Rb = 150e3;   % Ω
C1 = 4.7e-6; C2 = 0.22e-6;           % F
Rf = 47e3; R2 = 6.8e3;               % Ω
```

Estos valores son notablemente específicos — sugieren que ya había componentes físicos disponibles o que surgieron de un cálculo previo fuera del código (posiblemente a mano).

#### Función de transferencia del VCVS bandpass

```matlab
% Numerador
num = ((1 + (Rb/Ra)) / (R1 * C1)) * s;

% Denominador
den = s^2 + ((1/(R1*C1)) + (1/(R2*C1)) + (1/(R2*C2)) - (Rb)/(Ra*Rf*C1))*s + ...
      (R1 + Rf) / (R1 * Rf * R2 * C1 * C2);
```

La ganancia del amplificador VCVS es:
```
A_vcvs = 1 + Rb/Ra = 1 + 150k/10k = 16
```

El denominador tiene la forma clásica de un sistema de segundo orden:
```
s² + (2ζω₀)s + ω₀²
```

Donde el término de amortiguamiento incluye la resta de `(Rb)/(Ra*Rf*C1)` — esta es la realimentación positiva del VCVS que reduce el amortiguamiento y permite controlar Q.

#### La compensación del geófono integrada

El script también incluye el cálculo del compensador de geófono (el mismo principio que `compensadorKai.m` pero implementado en el propio circuito analógico):

```matlab
f0 = 10;          % Hz — frecuencia natural objetivo
wn = 2*pi*f0;
flow = 1;         % Hz — frecuencia de corte inferior
wlow = 2*pi*flow;
zeta_des = wn/(2*wlow);  % = 10/(2×1) = 5 → amortiguamiento crítico

H = 1 - 2*(zeta_des - 0.25)*wn*s / (s^2 + 2*zeta_des*wn*s + wn^2);
```

Con `f0=10Hz` y `flow=1Hz`:
- `wn = 2π×10 ≈ 62.83 rad/s`
- `zeta_des = 10/(2×1) = 5` (muy sobre-amortiguado — el compensador es agresivo)

La función `H` es la misma forma que `compensadorKai.m`: un "notch adaptado" que cancela el polo del geófono y lo reemplaza por uno más abierto. La diferencia es que aquí se estudia implementarlo en analógico con el VCVS.

#### Comparación con el enfoque anterior (análisis de tradeoffs)

```matlab
A = 1.2;
sys = (A - (91/3)*num/den) / A;   % Sistema combinado VCVS + compensación
```

El factor `91/3 ≈ 30.3` aparece como una escala empírica. La expresión `(A - K*H_vcvs)/A` sugiere que se está modelando el sistema donde el amplificador VCVS "recibe" una fracción de la señal compensada.

El script termina con un `bode(sys); hold on; bode(H)` para comparar visualmente la respuesta del circuito VCVS con el compensador ideal.

#### ¿Por qué explorar compensación analógica?

Al llegar a este punto del proyecto, hay tres opciones para mejorar la respuesta en baja frecuencia del geófono SM-24:

1. **Digital (MATLAB):** `compensadorKai.m` y `compensadorPazos.m` — funciona, pero solo en post-proceso
2. **Digital (PSoC FIR/DFB):** El filtro está en el microcontrolador — tiempo real, pero limitado por la resolución de coeficientes
3. **Analógico (VCVS):** Un circuito activo antes del ADC — tiempo real, sin carga computacional, pero requiere componentes físicos y calibración

La exploración de `Compensador.m` sugiere que se está evaluando la opción 3 como posibilidad para el diseño final del sistema. Si el geófono se puede compensar en analógico, el ADC ve directamente la señal de aceleración del suelo en todo el ancho de banda de interés, sin necesidad de procesamiento posterior.
