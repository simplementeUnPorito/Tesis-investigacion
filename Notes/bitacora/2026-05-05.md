### Desarrollo detallado: `analisis_espectral.m` — El framework científico

Este es el archivo más significativo del commit. En 264 líneas de MATLAB, se implementa una herramienta de análisis espectral que:

1. Lee datos de medición desde archivos `.mat` estructurados en `datos/`
2. Agrupa las mediciones por "entidad" (circuito analógico bajo test)
3. Computa PSD por el método de Welch
4. Calcula SNR en banda vs fuera de banda
5. Genera figuras A (PSD), B (SNR), y C (espectrogramas)

#### El concepto de "entidad"

El script introduce el concepto de **entidad** como unidad organizacional:

```matlab
ent = struct('nombre','','fs',1020,'fMin',0,'fMax',0,'ganancia',1,'observ','');
if isfield(d,'entidad'), ent = d.entidad; end
```

Cada archivo `.mat` puede tener metadata de entidad:
- `nombre`: identificador del circuito (ej. "MFB", "Sallen-Key", "diferencial")
- `fs`: frecuencia de muestreo real
- `fMin`, `fMax`: banda de interés para calcular SNR
- `ganancia`: factor de conversión a mV

La existencia de `datos/MFB_DigitalFilterLPF.mat` y `datos/Sallen_Key.mat` confirma que se midieron dos circuitos diferentes con esta metodología.

#### El método Welch con resolución geofísica

```matlab
WIN_WELCH    = 2^14;   % 16384 muestras → Δf ≈ 0.062 Hz a 1020 SPS
OVL_WELCH    = 2^13;   % 50% solapamiento
NFFT_WELCH   = 2^14;
F_BANDA_BAJA = 0.1;    % Hz
F_BANDA_MED  = 100;    % Hz
```

La ventana de Welch de **16384 muestras** a 1020 SPS requiere:
```
16384 / 1020 ≈ 16.06 segundos de señal por segmento
```

Con 50% de solapamiento, para tener al menos 2 segmentos se necesitan ≥24 segundos de captura. Esto explica por qué algunas sesiones del 4 de mayo eran cortas (buffer agotado antes de llegar al mínimo).

La resolución espectral de **0.062 Hz** es notable:
- El SM-24 tiene frecuencia natural fn ≈ 4.5 Hz
- Para MASW necesitás resolver curvas de dispersión en 1-100 Hz
- 0.062 Hz permite identificar picos individuales en el espectro con alta precisión

#### El cálculo de SNR

```matlab
function [snr_dB, pxx, f] = calcPSD(seg, fs, fn, win, ovl, nfft, flo, fhi)
    seg = seg - mean(seg);   % elimina DC antes del PSD
    [pxx, f] = pwelch(seg, hann(win), ovl, nfft, fs);
    df = f(2) - f(1);
    f_start = df;            % excluye bin DC (f=0)
    inband  = f >= flo & f <= fhi;
    outband = f >= f_start & ~inband;
    E_in  = mean(pxx(inband));
    E_out = mean(pxx(outband));
    snr_dB = 10*log10(E_in / (E_out + eps));
end
```

El SNR se define como la relación entre la densidad espectral media **dentro de la banda de interés** y la densidad espectral media **fuera de la banda**:
- SNR > 0 dB → hay más señal en banda que fuera de banda (buen filtro)
- SNR < 0 dB → más energía fuera de banda (mal filtro o señal no está en banda)

La sustracción de media antes del Welch (`seg = seg - mean(seg)`) elimina el offset DC que de otro modo dominaría el bin f=0 y contaminaría el `outband` calculado.

**Figura A — PSD:** Dos subplots (raw vs filtrada MATLAB). Cada medición en gris traslúcido, media±σ en color sólido. Bandas de interés resaltadas en verde. Escala logarítmica en frecuencia. Esto permite comparar visualmente el ruido de fondo y la respuesta espectral de cada topología.

**Figura B — SNR por medición:** Barras para cada sesión de captura, con líneas horizontales de la media. Permite identificar qué sesiones fueron más representativas (SNR más estable) y descartar outliers.

### Desarrollo detallado: `filtroFir.m` — El filtro multibanda

El script `filtroFir.m` reemplaza el enfoque de notch puro de `notch.m` con un diseño multibanda más sofisticado:

```matlab
% Bandas:
%   0 – 45 Hz  : pasa (señal sísmica)
%  49 – 51 Hz  : notch (50 Hz red eléctrica)
%  55 – 90 Hz  : pasa (señal sísmica alta)
% 100 – 510 Hz : corte (anti-aliasing)

coeficientes_b = firpm(800, [0 100, 110 510]/510, [1 1, 0 0], ...
    [1, 1/(10^(-200/20))]);
```

**Orden 800** — esto es enormemente más grande que el notch de 464 taps. El filtro de 800 taps tiene:
- Bandas de transición angostas (entre 45-49 Hz y 51-55 Hz)
- 40 dB de atenuación en 50 Hz
- 200 dB de atenuación sobre 110 Hz

La atenuación de 200 dB en la banda de rechazo superior es prácticamente infinita (los números de punto flotante tienen solo ~308 dB de rango dinámico). Esto garantiza que el anti-aliasing sea perfecto.

El script incluye comentarios que muestran la evolución del diseño — el código comentado al principio era el notch simple de `notch.m`, reemplazado por la especificación multibanda más completa.

`fvtool(coeficientes_b, 1, 'Fs', fs)` — MATLAB's Filter Visualization Tool abre automáticamente para inspeccionar la respuesta en frecuencia.

### PCB_Piezotronics_086D20_Manual.pdf

El vault de Obsidian gana un nuevo documento: el manual del **PCB Piezotronics Model 086D20**, un martillo de impacto con sensor de fuerza piezoeléctrico (Piezoelectric Impulse Hammer).

Este es el instrumento clásico para hacer MASW activo:
1. El martillo golpea el suelo y genera ondas Rayleigh
2. El geófono SM-24 registra la propagación de esas ondas
3. La diferencia de tiempo de llegada entre múltiples geófonos permite calcular la velocidad de ondas de cizalla (Vs)

La aparición del manual del martillo sugiere que se está pensando en el sistema completo de adquisición MASW. No solo el geófono receptor, sino también la fuente activa de señal.

**Especificaciones del PCB 086D20:**
- Tipo: Impulse Hammer (martillo de impacto)
- Sensor: Piezoeléctrico integrado
- Rango de fuerza: ±500 lbf (tipicamente)
- Uso: Pruebas de impacto modal, MASW activo

Agregar este manual al vault es un movimiento consciente: se está documentando los instrumentos del sistema MASW completo para la tesis.

Cuatro PGAs: dos negativos y dos positivos. Esto corresponde a una **configuración de dos etapas de ganancia** para cada canal del amplificador diferencial:

```
SM-24 (+) → Buffer_p → PGA_p_1 → PGA_p_2 → 
                                              } → OpaDiff → ADC_DelSig
SM-24 (-) → Buffer_n → PGA_n_1 → PGA_n_2 →
```

Dos etapas de PGA permiten ganancias más altas con mejor linearidad que una sola etapa de ganancia alta. Si cada PGA puede hacer ×8, dos en cascada dan ×64 sin saturación prematura.
