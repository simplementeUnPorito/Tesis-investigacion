#### Por qué un PSoC separado para el martillo

El martillo de impacto PCB Piezotronics 086D20 es un instrumento de medición activo: tiene un piezoeléctrico que genera una señal de voltaje proporcional a la fuerza del golpe. El rango de salida es ±5V (o similar), y la señal tiene características muy diferentes al geófono SM-24:

- **Duración del pulso**: 1–10 ms (el geófono genera ondas que duran hasta 1-2 segundos)
- **Frecuencia dominante**: depende de la punta (170–1600 Hz), mucho más alta que las ondas Rayleigh (0.5–50 Hz)
- **Amplitud**: mayor y más variable (depende de la fuerza del golpe)
- **Ruido**: diferente perfil de ruido (el martillo está más cerca de la fuente, menos ruido de suelo)

Intentar leer el martillo con el mismo PSoC que el geófono crea problemas:
1. La cadena analógica (OPAbp + OPAadder + OPAlp) está diseñada para el geófono, no para el martillo
2. Las frecuencias de muestreo óptimas son diferentes
3. El ADC Delta-Sigma del proyecto geófono tiene latencia de pipeline que distorsiona la detección del onset del golpe

La solución: un PSoC dedicado al martillo con su propia cadena de acondicionamiento.

#### `AcondicionamientoMartillo/Design01.cydsn` — la cadena del martillo

El `main.c` del 28 de mayo (versión inicial) inicializa:

```c
#include "project.h"

CY_ISR(getADCdata){
    // VACÍO — pendiente implementar
}

int main(void)
{
    CyGlobalIntEnable;
    Opa_ref_Start();     // Referencia de tensión
    VDAC_Start();        // VDAC para offset
    Opa_buff_1_Start();  // Buffer op-amp 1
    Opa_buff_2_Start();  // Buffer op-amp 2
    Opa_buff_3_Start();  // Buffer op-amp 3
    PGA_1_Start();       // PGA ajustable
    ADC_Start();         // SAR ADC
    isr_ADC_StartEx(getADCdata);
    LPF_1_Start();       // LPF analógico 1
    LPF_2_Start();       // LPF analógico 2

    for(;;) {
        /* Pendiente */
    }
}
```

El proyecto tiene dos ADCs presentes en el código generado:
- `ADC` (SAR) — para el muestreo rápido del martillo (~4000 SPS o más)
- `ADC_DelSig` — posiblemente para el canal de referencia
- `ADC_AMux` — multiplexor analógico para seleccionar la fuente

Los tres buffers (`Opa_buff_1/2/3`) sugieren que la señal del martillo pasa por una etapa de buffer antes del PGA. Los buffers de op-amp presentan alta impedancia de entrada (para no cargar el piezoeléctrico del martillo) y baja impedancia de salida (para conducir la cadena siguiente).

La ISR `getADCdata` está vacía en esta versión — el proyecto está en etapa de configuración de hardware, aún sin lógica de software. El "AYUDAAAAAAAAAAA" indica que incluso poner en marcha este esqueleto de proyecto fue difícil (probablemente conflictos entre los bloques analógicos del PSoC — el PSoC 5LP tiene un número limitado de op-amps y su asignación a pines específicos puede causar conflictos con otros proyectos).

#### Cambios en `platformio.ini` (master y slave)

Los `platformio.ini` también fueron modificados — seguramente para desactivar el stub del martillo (`HAMMER_IMU_ENABLED=0`) ahora que hay hardware real en desarrollo, o para ajustar los flags de debug ahora que el PSoC del martillo tiene su propio hardware que mapea a pines distintos.

#### El problema no enunciado

El commit no dice qué fue lo que causó el "AYUDAAAAAAAAAAA". Las posibilidades basadas en el contexto:

1. **El sistema del 26 dejó de funcionar**: Entre el 26 y el 28, algo cambió en el firmware (recompilación del PSoC, reflash del ESP32) que rompió lo que funcionaba. Bugs de sincronización en sistemas distribuidos a menudo son no determinísticos — funcionan a veces y no otras.

2. **Conflicto de recursos en el PSoC**: Al intentar agregar el `AcondicionamientoMartillo`, se descubrió que los bloques analógicos disponibles (OA0–OA3) ya estaban todos usados por el `DiferencialToSingleEnded_ESP`. El PSoC 5LP tiene exactamente 4 op-amps — no hay margen.

3. **Problema de sincronización GPIO**: El pin SYNC del maestro ESP32 que va al PSoC del geófono necesita ahora ir también al PSoC del martillo para sincronizar ambas capturas. El cableado de tres dispositivos (ESP32 + PSoC geófono + PSoC martillo) introduce más puntos de falla.

El "AYUDAAAAAAAAAAA" es la frustración de no saber exactamente cuál de estos es el problema — o de que son todos simultáneamente.
