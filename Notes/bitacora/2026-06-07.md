## Contexto del día

El sistema MASW llegó al domingo 7 de junio en un estado de relativa estabilidad operativa. Los commits anteriores (5 de junio) habían dejado el proyecto PSoC `AcondicionamientoAnalogico` reconstruido, la detección automática por `TopDesign` funcionando, y la GUI Python con el layout de plots corregido. La gran pregunta que quedaba abierta era: ¿cómo sabe el software en PC cuál es la frecuencia de muestreo real del ADC?

Este es un problema que suena trivial pero es engañosamente profundo en sistemas embebidos heterogéneos. El PSoC 5LP con su `ADC_DelSig` tiene una frecuencia de muestreo que depende del divisor de clock configurado en PSoC Creator — una constante de compilación, no algo que se puede interrogar en tiempo de ejecución desde afuera. Hasta hoy, tanto la GUI Python como MATLAB usaban `config.FS` (un valor hardcodeado estático). Eso funciona siempre que el firmware PSoC y la configuración Python estén en sincronía perfecta, pero basta cambiar el proyecto PSoC Creator (reclocking, cambio de decimation rate del Delta-Sigma) y el software de PC queda desincronizado silenciosamente.

La solución elegida este día: hacer que el PSoC **reporte su propia frecuencia de muestreo** a través del canal UART existente, propagarla por ESP-NOW hasta el nodo maestro, y de ahí a MATLAB/Python a través del protocolo serie sobre USB. El campo `sample_rate` en el mensaje `HELLO` pasa a ser el vehículo de este auto-reporte.

El segundo gran cambio del día es la sustitución del cancelador notch LMS (Least Mean Squares, adaptivo muestra a muestra) por un ajuste de mínimos cuadrados sobre el buffer completo de captura. Esta decisión tiene motivaciones técnicas sólidas que se analizan abajo.

Finalmente, una limpieza de vault de Obsidian: mover las imágenes pegadas a una subcarpeta `Assets/` y renombrar el archivo base.

#### Diff y análisis técnico

##### 1 — Nuevo protocolo: `PSOC_CTRL_FS_REPORT` (0xC3)

El primer cambio está en el firmware PSoC y define un nuevo tipo de frame en el canal UART PSoC→ESP:

```c
// src/psoc/AcondicionamientoAnalogico.cydsn/main.c
static void uart_send_fs_report(void)
{
    uint8 fs_lo = (uint8)((uint16)ADC_DEFAULT_SRATE & 0xFFu);
    uint8 fs_hi = (uint8)(((uint16)ADC_DEFAULT_SRATE >> 8u) & 0xFFu);
    uint8 frame[5u];
    frame[0] = 0xABu;      // header estándar del protocolo
    frame[1] = 0xC3u;      // nuevo opcode: FS_REPORT
    frame[2] = fs_lo;
    frame[3] = fs_hi;
    frame[4] = (uint8)(0xC3u ^ fs_lo ^ fs_hi);  // CRC XOR simple
    UART_PutArray(frame, (uint8)sizeof(frame));
}
```

Este frame se activa inmediatamente después del `PING` de respuesta (`uart_send_ping()`), cuando el ESP hace la sonda de detección con el comando `0xA5u`:

```c
case 0xA5u:
    uart_send_ping();
    uart_send_fs_report();   // NUEVO: reportar FS junto al ping
    break;
```

`ADC_DEFAULT_SRATE` es una constante del proyecto PSoC Creator que refleja la configuración real del `ADC_DelSig`. Al enviarla como dos bytes little-endian con CRC XOR, el ESP esclavo puede leerla y propagarla hacia arriba en la cadena.

El mismo cambio se aplicó al proyecto `Martillo_ESP.cydsn/main.c` (el firmware del nodo martillo), manteniendo ambos proyectos PSoC sincronizados en el protocolo.

##### 2 — ESP esclavo: parsing del FS_REPORT

En `psoc_uart.h` se agrega la constante y el estado:

```cpp
// src/esp/Nodo comunicación/slave/src/psoc_uart.h
#define PSOC_CTRL_FS_REPORT 0xC3  /* PSoC → ESP: frecuencia de muestreo ADC */

// En la clase PsocUART:
uint16_t sampleRate() const { return _sampleRate; }
// ...
private:
    uint16_t _sampleRate = 0;   // 0 = no reportado aún
    void _parseFsReport();
```

Y en `psoc_uart.cpp` se implementa el parser, integrado en el loop de polling UART:

```cpp
// src/esp/Nodo comunicación/slave/src/psoc_uart.cpp
void PsocUART::_parseFsReport()
{
    const uint8_t fs_lo = _buf[2];
    const uint8_t fs_hi = _buf[3];
    const uint8_t crc   = _buf[4];
    if (crc != (uint8_t)(PSOC_CTRL_FS_REPORT ^ fs_lo ^ fs_hi)) {
        _badLen++;
        return;
    }
    const uint16_t fs = (uint16_t)fs_lo | ((uint16_t)fs_hi << 8);
    if (fs > 0) { _sampleRate = fs; }
}
```

El frame corto de 5 bytes es despachado en el `poll()` principal:

```cpp
} else if (_buf[1] == PSOC_CTRL_FS_REPORT) {
    continue;   /* frame corto: [AB][C3][fs_lo][fs_hi][crc] */
}
// ...
if (_idx >= PSOC_CTRL_ACK_BYTES && _buf[1] == PSOC_CTRL_FS_REPORT) {
    _idx = 0;
    _parseFsReport();
    continue;
}
```

La arquitectura del parser es state-machine basada en longitud de frame: el código primero detecta que el segundo byte es `0xC3` cuando todavía tiene datos parciales (para evitar interpretarlo como un frame de datos largo), y luego parsea cuando tiene los 5 bytes completos.

##### 4 — Codificación compacta en el paquete MATLAB

El protocolo serie USB maestro→PC usa paquetes fijos de 6 bytes. Para no romper el formato, el campo `sample_rate` se codifica como `fs/100` en el byte `b0` del paquete de notificación HELLO:

```cpp
// src/esp/Nodo comunicación/master/src/matlab_transport.h
inline void MatlabTransport::sendHelloNotif(uint8_t nodeId, uint8_t psoc_ok, 
                                             const uint8_t mac[6], uint16_t fs)
{
    /* b0=fs/100 (0=desconocido, 1=100Hz, 8=800Hz, 10=1000Hz...) */
    uint8_t fs_code = (fs >= 100u) ? (uint8_t)(fs / 100u) : 0u;
    uint8_t pkt[6] = { MATLAB_PKT_HEADER, nodeId, 0xFD, 0x01, psoc_ok, fs_code };
    Serial.write(pkt, 6);
    // ... envío de MAC en 3 sub-paquetes adicionales
}
```

La escala de `fs/100` en un `uint8_t` cubre frecuencias de 100 Hz a 25 500 Hz — más que suficiente para cualquier ADC de geofísica sísmica. El valor 0 significa "no reportado aún". En el lado Python:

```python
# src/python/geophone_scope/protocol.py
@property
def hello_fs_hz(self) -> int:
    """Sample rate reported in a slave HELLO packet (b0 = fs/100 Hz, 0 = unknown)."""
    return self.b0 * 100
```

Simple y elegante: multiplica por 100 para revertir la compresión.

##### 6 — Reemplazo del cancelador LMS por mínimos cuadrados

Este es el cambio algorítmico más significativo del día. En `signal_proc.py`, la función `notch_canceller` (LMS adaptivo muestra a muestra) es reemplazada por `harmonic_notch` (ajuste por mínimos cuadrados sobre el buffer completo):

**Antes — LMS adaptivo:**
```python
def notch_canceller(x, w, fs, f0, n_harmonics, mu):
    """
    LMS adaptive notch filter that cancels f0 and its harmonics.
    Filter weights are adapted sample-by-sample using the Widrow–Hoff LMS rule.
    """
    n_weights = 2 * n_harmonics
    if w is None or len(w) != n_weights:
        w = np.zeros(n_weights, dtype=np.float64)
    
    n     = len(x)
    y_out = np.empty(n, dtype=np.float64)
    # ... pre-computa sinusoides de referencia ...
    
    w = w.copy()
    for i in range(n):          # loop muestra a muestra
        ref_i     = refs[:, i]
        noise_est = float(np.dot(w, ref_i))
        e         = float(x[i]) - noise_est
        y_out[i]  = e
        w        += 2.0 * mu * e * ref_i   # actualización LMS
    
    return y_out, w             # devuelve también los pesos actualizados
```

**Después — Mínimos cuadrados batch:**
```python
def harmonic_notch(x, fs, f0, n_harmonics):
    """
    Remove f0 and its harmonics from x with a one-shot least-squares fit.
    
    Mains hum sits at a known, very stable frequency, so once the whole
    capture is available this direct fit is both exact (optimal in the
    least-squares sense) and instantaneous — unlike LMS/NLMS, it needs no
    step size and has no convergence transient or steady-state misadjustment.
    """
    x = np.asarray(x, dtype=np.float64)
    n = len(x)
    if n == 0 or n_harmonics <= 0:
        return x.copy()
    
    t = np.arange(n, dtype=np.float64)
    cols = []
    for k in range(1, n_harmonics + 1):
        angle = 2.0 * np.pi * (k * f0) / fs * t
        cols.append(np.cos(angle))
        cols.append(np.sin(angle))
    h = np.column_stack(cols)   # matriz de diseño: n × (2*n_harmonics)
    
    coeffs, *_ = np.linalg.lstsq(h, x, rcond=None)
    return x - h @ coeffs       # señal menos la interferencia ajustada
```

La diferencia conceptual es profunda. El LMS adaptivo actualiza sus pesos `w` muestra a muestra usando la regla de Widrow-Hoff: `w += 2μ·e·r`, donde `e` es el error y `r` es la referencia sinusoidal. Esto tiene dos problemas para un notch de frecuencia de red conocida:

1. **Transitorio de convergencia**: Los primeros N/µ muestras tienen cancelación degradada mientras los pesos convergen. Con `µ = 0.002` (valor anterior por defecto), la convergencia tarda cientos de muestras.

2. **Ajuste del step size µ**: El usuario necesita sintonizar µ manualmente. Demasiado grande → inestabilidad; demasiado pequeño → convergencia lenta. Era el spinbox "µ:" que se eliminó de la UI.

El enfoque de mínimos cuadrados batch evita ambos problemas. Dado que conocemos exactamente la frecuencia fundamental (50 Hz de red), construimos la matriz de diseño `H` con pares cos/sin para cada armónico y resolvemos `H·c = x` en el sentido de mínimos cuadrados con `np.linalg.lstsq`. El resultado es la estimación óptima (en norma L2) de la interferencia armónica en el buffer completo. Restar esa estimación da la señal limpia.

El costo: requiere el buffer completo, no puede operar en streaming muestra a muestra. Pero para MASW — donde se captura un registro completo de 1-5 segundos antes de analizarlo — esto es exactamente lo que se quiere.

##### 7 — `_reprocess_filt_buf`: unificación del pipeline de señal

En `main_window.py` se introduce un método central que reconstruye `filt_buf` desde cero cada vez que cambia algún parámetro del pipeline:

```python
def _reprocess_filt_buf(self, ch_index: int) -> None:
    """Rebuild filt_buf from raw_buf: FIR -> DC removal -> harmonic notch."""
    nd = self._data[ch_index]
    nd.filt_buf.clear()
    if not nd.raw_buf:
        nd.filt_zi = None
        return
    
    raw = np.array(nd.raw_buf, dtype=np.float64)
    if nd.filt_b is not None:
        arr, nd.filt_zi = signal_proc.fir_filter(nd.filt_b, raw, None)
    else:
        arr = raw.copy()
        nd.filt_zi = None
    
    if nd.dc_remove and len(raw) > 1:
        arr = arr - float(np.mean(raw))
    
    if nd.notch_enabled:
        arr = signal_proc.harmonic_notch(arr, nd.fs, config.NOTCH_F0, nd.notch_harm)
    
    nd.filt_buf.extend(arr.astype(np.float32).tolist())
```

Este método se llama desde: `_on_fir_apply`, `_on_fir_remove`, `_on_notch_toggled`, `_on_notch_harm`. Y durante el streaming, si el notch está activado, se recalcula en cada tick del render timer:

```python
def _render(self) -> None:
    for i, nd in enumerate(self._data.nodes):
        # The harmonic notch fits over the *complete* capture, so while
        # streaming with it enabled, keep refitting it to the growing buffer.
        if nd.notch_enabled and self._streaming:
            self._reprocess_filt_buf(i)
```

Esto asegura que al ir llegando muestras, el notch se re-ajusta progresivamente al buffer creciente.

def _effective_fs(self) -> float:

#### Impacto en el proyecto

Este commit completa el "circuito de retroalimentación de metadatos": el PSoC reporta su propia configuración de hardware (frecuencia de muestreo) al software de PC, que deja de depender de una constante hardcodeada. El sistema pasa de ser **opaco** (el software asume que sabe cómo está configurado el hardware) a **auto-descriptivo** (el hardware informa sus parámetros de operación).

El reemplazo del notch LMS por mínimos cuadrados elimina un parámetro de usuario (µ) y un modo de falla (convergencia lenta o inestabilidad). Para geofísica sísmica, donde la fuente de interferencia (red eléctrica a 50 Hz) es exactamente conocida y estable, el enfoque analítico directo es superior al adaptivo en todas las métricas relevantes: calidad de cancelación, robustez, y simplicidad de uso.

El tema central del día fue la **propagación de metadatos de hardware**: hacer que el PSoC le diga al mundo cuántas muestras por segundo está tomando. Es el tipo de feature que parece pequeño pero tiene implicaciones sistémicas grandes. Antes, cambiar la frecuencia de muestreo en PSoC Creator requería también actualizar `config.FS` en Python y el equivalente en MATLAB — dos lugares adicionales donde el error podía colarse. Ahora la verdad viene del hardware.

El reemplazo del notch LMS por mínimos cuadrados es una decisión de madurez técnica. El LMS es una solución elegante para señales no estacionarias con frecuencias desconocidas, pero para el ruido de red eléctrica (50 Hz exactos, muy estable) es overkill complicado. El enfoque de mínimos cuadrados sobre el buffer completo es el correcto para este dominio: exacto, sin hiperparámetros, sin transitorios, óptimo en el sentido L2.

La adición de los símbolos BNC, BoostConverter y Geophone en PSoC Creator completa la narrativa del front-end analógico: el esquemático ahora representa visualmente todos los componentes del nodo — no solo los bloques programables PSoC sino también el hardware de interfaz con el mundo real.

El sistema MASW, al final de este domingo, tiene su pila de comunicación más robusta que nunca: el PSoC sabe quién es, el ESP sabe qué tiene conectado, MATLAB y Python saben con qué frecuencia están digitalizando. Todos los niveles del stack ahora hablan el mismo idioma.
