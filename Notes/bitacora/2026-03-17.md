Este es el primer script MATLAB propio del proyecto. Es una función de inicialización del entorno:

```matlab
function project_root = init_project()
%INIT_PROJECT Configura el path del proyecto Tesis para MATLAB.
%
% Agrega al path:
%   - src/matlab
%   - third_party/MASW-Matlab-code
%
% Devuelve:
%   project_root -> ruta absoluta a la raíz del proyecto

    % Ruta absoluta de este archivo
    this_file = mfilename('fullpath');
    this_dir  = fileparts(this_file);

    % this_dir = .../Tesis/src/matlab
    % subir dos niveles hasta .../Tesis
    project_root = fileparts(fileparts(this_dir));

    matlab_src = fullfile(project_root, 'src', 'matlab');
    masw_path  = fullfile(project_root, 'third_party', 'MASW-Matlab-code');

    % Verificaciones mínimas
    if ~isfolder(matlab_src)
        error('init_project:MissingFolder', ...
            'No existe la carpeta MATLAB esperada: %s', matlab_src);
    end

    if ~isfolder(masw_path)
        warning('init_project:MissingThirdParty', ...
            'No existe la carpeta third_party esperada: %s', masw_path);
    end

    % Agregar paths evitando duplicación innecesaria
    addpath(genpath(matlab_src));

    if isfolder(masw_path)
        addpath(genpath(masw_path));
    end

    fprintf('Proyecto inicializado correctamente.\n');
    fprintf('Root: %s\n', project_root);
end
```

Este script muestra buenas prácticas de MATLAB:
- Usa `mfilename('fullpath')` para detectar dónde está él mismo (independiente del directorio actual)
- Usa `fileparts()` dos veces para subir dos niveles en la jerarquía de directorios
- Tiene validaciones con `isfolder()` antes de agregar al path
- Distingue entre `error()` (falta el src propio, fallo crítico) y `warning()` (falta el third_party, fallo no crítico)
- Usa `genpath()` para agregar recursivamente todas las subcarpetas al path de MATLAB

El propósito: desde cualquier sesión de MATLAB, correr `init_project()` al inicio establece el entorno completo. Sin esto, habría que agregar paths manualmente cada vez — error-prone y tedioso.

Los scripts Python `fill_wave_foundations.py`, `fix_dispersion_layer.py`, `reorder_concepts.py` y `.concept_reorder_manifest.json` se reorganizan de la raíz y `src/` a `src/python/Ordenar Obsidian/`, estableciendo la estructura de carpetas final de los scripts de utilidad.

```markdown
### 2.2.2 Derivación mediante potenciales de onda elástica

La existencia de las [[Rayleigh Waves]] se demuestra formalmente mediante la 
descomposición de Helmholtz del campo de desplazamientos, aplicando condiciones 
de frontera en la superficie libre.

#### Planteamiento del problema

Se considera un medio:
- elástico lineal / homogéneo / isotrópico
- semi-infinito: z ≥ 0 (superficie libre en z = 0)

El campo de desplazamientos u se descompone mediante los [[Elastic Wave Potentials]]:

u = ∇φ + ∇×ψ

donde:
- φ → potencial escalar (asociado a [[P-waves]])
- ψ → potencial vectorial (asociado a [[S-Waves]])

Cada potencial satisface su propia ecuación de onda:
∇²φ = (1/VP²) ∂²φ/∂t²
∇²ψ = (1/VS²) ∂²ψ/∂t²
```

Esta derivación es el corazón de la teoría de ondas de Rayleigh. La descomposición de Helmholtz separa el campo de desplazamientos en una parte irrotacional (potencial P) y una parte sin divergencia (potencial S). Las condiciones de frontera en z=0 (esfuerzo superficial = 0) luego determinan la relación entre los potenciales, que produce la ecuación de dispersión de Rayleigh.

```markdown
#### Solución armónica en 2D

Para el problema plano (propagación en x, profundidad z):

φ = A · e^(-αz) · e^(i(kx - ωt))
ψ = B · e^(-βz) · e^(i(kx - ωt))

donde:
α² = k² - ω²/VP² = k² - kP²
β² = k² - ω²/VS² = k² - kS²

Para ondas que decaen con la profundidad (ondas superficiales): α, β > 0
```

Esta es la condición clave: para que existan ondas superficiales, la amplitud debe decaer con la profundidad. Eso requiere que α y β sean reales y positivos, lo cual implica que k > kP y k > kS — es decir, la velocidad de fase de Rayleigh debe ser menor que VS.

```markdown
## El Problema de Lamb (1904)

El Problema de Lamb describe la generación de ondas elásticas producidas por 
una carga puntual aplicada sobre la superficie de un medio elástico semi-infinito.

### Fundamento físico
Cuando una fuerza impulsiva se aplica en la superficie, se generan simultáneamente:
- [[P-waves]] — se propagan hacia el interior en 3D
- [[S-Waves]] — se propagan hacia el interior en 3D  
- [[Rayleigh Waves]] — se propagan a lo largo de la superficie

Las ondas de cuerpo decaen en superficie como:
u_body ∝ 1/r²

Las ondas de Rayleigh decaen como:
u_R ∝ 1/√r

### Aplicación a geófonos
La transición al régimen dominado por Rayleigh ocurre a:
r ≳ 1–2λ

Esto justifica la regla práctica: el primer geófono debe ubicarse a distancia 
mínima ≥ λ_max/2 de la fuente para evitar la zona de campo cercano.
```

Este concepto tiene aplicación práctica directa: al diseñar el arreglo de geófonos del sistema de adquisición, hay que respetar esta distancia mínima. Si el primer geófono está demasiado cerca de la fuente, las ondas de cuerpo (que todavía no han atenuado) contaminan la señal y distorsionan la curva de dispersión.

**`Elastic Wave Potentials.md` (83 líneas)** — la descomposición de Helmholtz:

```markdown
La descomposición de Helmholtz establece que cualquier campo vectorial F 
puede expresarse como:

F = ∇φ + ∇×ψ

donde φ es el potencial escalar irrotacional y ψ el potencial vectorial 
solenoidal. Aplicado al campo de desplazamientos elásticos:

- φ → satisface la ecuación de onda de P: ∇²φ = (1/VP²)φ̈
- ψ → satisface la ecuación de onda de S: ∇²ψ = (1/VS²)ψ̈

Esta separación es fundamental porque permite resolver el problema de P y S 
independientemente antes de aplicar las condiciones de frontera.
```

**`Mode Superposition.md` (49 líneas)** — suma de modos de Rayleigh:

En medios estratificados, la curva de dispersión puede tener múltiples **modos**. El modo fundamental (el de mayor longitud de onda) suele dominar, pero los modos superiores también contribuyen a la señal. En el procesamiento MASW estándar solo se identifica el modo fundamental, pero la superposición de modos puede complicar la identificación.

**`Rayleigh Eigenproblem.md` (55 líneas)** — el problema de autovalores:

La ecuación de dispersión de Rayleigh en un medio estratificado es un **problema de autovalores**: para cada frecuencia ω, se buscan los valores de k (autovalores) que satisfacen las condiciones de frontera en todas las interfaces. El método de las matrices de Thomson-Haskell (que aparece más adelante en el vault) es el algoritmo estándar para resolver este problema.

**`Viscoelastic Media.md` (78 líneas)** — la extensión viscoelástica de Foti:

```markdown
## Número de onda complejo en medios viscoelásticos

En medios con disipación, el número de onda k se vuelve complejo:
k̃ = kR + i·kI

El término kI controla el decaimiento exponencial:
u ∝ e^(-kI x)

El material damping ratio D cuantifica la fracción de energía disipada:
D = 2·kI/kR  (para amortiguamiento pequeño)

En el dominio de la frecuencia, los módulos elásticos se reemplazan por 
módulos complejos dependientes de la frecuencia:
G*(ω) = G(ω)(1 + 2iD(ω))
```

Este es el punto de conexión entre la teoría elástica (ideal) y la realidad: los suelos reales amortiguan las ondas. La viscoelasticidad lineal es el framework matemático que permite modelar ese amortiguamiento.
