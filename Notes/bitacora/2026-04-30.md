### Desarrollo detallado: `ComparacionFiltros.m` — El análisis que dictaminó el pivot

Este script de 64 líneas es la evidencia del análisis técnico que llevó al cambio de topología. Compara tres funciones de transferencia:

```matlab
% Ecuación 1 — MFB (Multiple Feedback, pasa-banda)
R1_1 = 82e3;   R2_1 = 750e3;   R5_1 = 82e3;
C3_1 = 44e-6;  C4_1 = 1e-9;

Num1 = -s * (1 / (R1_1 * C4_1));
Den1 = s^2 + s * ((C3_1 + C4_1) / (C3_1 * C4_1 * R5_1)) + ...
       (1 / (R5_1 * C3_1 * C4_1)) * (1/R1_1 + 1/R2_1);
H1 = Num1 / Den1;

% Ecuación 2 — VCVS/Sallen-Key (de Compensador.m anterior)
R1_2 = 330e3;  R2_2 = 91e3;   Rf_2 = 2e6;
Ra_2 = 40e3;   Rb_2 = 120e3;

Num2 = (1 + Rb_2/Ra_2) * (s / (R1_2 * C1_2));
Den2 = s^2 + s * (...) + (...);

% Ecuación 3 — Compensación de geófono (forma analítica)
gamma1 = 100;   gamma0 = 0.25;   w0 = 2*pi*10;
H3 = 2*(gamma1 - gamma0)*w0*s / (s^2 + 2*gamma1*w0*s + w0^2);
```

#### ¿Por qué MFB sobre Sallen-Key?

La topología **Sallen-Key** (también llamada VCVS) usa un amplificador operacional en modo seguidor de tensión con realimentación positiva para reducir el amortiguamiento. El problema en la práctica:

1. **Sensibilidad a tolerancias**: La realimentación positiva amplifica los errores de los componentes. Un condensador con tolerancia del 10% puede cambiar dramáticamente el Q del filtro.
2. **Estabilidad**: La ganancia >1 del VCVS puede llevar a oscilación si los componentes no son exactos.
3. **Impedancia de salida**: El VCVS ideal tiene impedancia de salida cero, pero el op-amp real tiene impedancia no nula que afecta la respuesta al conectar cargas.

La topología **MFB** (Multiple Feedback) usa el amplificador en modo inversor con realimentación negativa exclusivamente:

```
               C4
          ┌────┤├────┐
          │          │
R1   R2   │   (-)    │
●──┤├──┤├─┤   Op-Amp ├──● salida
          │   (+)    │
         C3   GND

Nodo suma: R5 entre nodo central y GND
```

Ventajas del MFB:
1. **Solo realimentación negativa**: El amplificador siempre es estable
2. **Menos sensible a tolerancias**: Los errores de componentes no se amplifican
3. **Ganancia inversora**: La señal sale invertida (signo −) pero eso es irrelevante para señales sísmicas donde solo importa la amplitud

La función de transferencia del MFB pasa-banda de segundo orden:

```
H1(s) = -s/(R1·C4) / (s² + s·(C3+C4)/(C3·C4·R5) + (1/(R5·C3·C4))·(1/R1+1/R2))
```

Con los valores elegidos:
- Componentes MFB: R1=82kΩ, R2=750kΩ, R5=82kΩ, C3=44µF, C4=1nF
- Frecuencia central: ω₀ = √[(1/R5·C3·C4)·(1/R1+1/R2)]

La combinación C3=44µF y C4=1nF es inusual — una diferencia de 44,000× entre los dos condensadores. Este diseño permite ajustar independientemente la frecuencia central y el Q del filtro.

El script termina con:

legend('Ecuación 1 (MFB)', 'Ecuación 2 (VCVS)', 'Ecuación 3 (Compensación)');
```

Al trazar los tres Bode superpuestos, queda claro cuál de los circuitos se acerca más a la respuesta de compensación ideal (H3). El script era la herramienta de decisión.

#### La "suma en corriente" (TIA)

El segundo punto del commit — "suma en corriente lo que es fachero" — se refiere al uso de amplificadores de transimpedancia (TIA) como etapa de suma. En el circuito diferencial del PSoC ya había `TIA_n` y `TIA_p` como componentes de firmware.

La suma en corriente funciona así: múltiples señales de voltaje se aplican a través de resistencias a un nodo virtual de tierra del amplificador. El TIA convierte la suma de corrientes en voltaje de salida. Es un sumador preciso, lineal, y no carga las fuentes de señal. "Fachero" porque evita el uso de amplificadores de instrumentación pesados y permite escalar el circuito fácilmente.

¿Por qué el estacionamiento? Es un lugar con múltiples fuentes de vibración predecibles:
- Autos pasando generan ondas elásticas en el asfalto
- Personas caminando producen señal de baja frecuencia
- El suelo del estacionamiento tiene una respuesta conocida (asfalto/hormigón)
- Está cerca del laboratorio pero es exterior, lo que elimina el ruido de equipos eléctricos

Los archivos `.mat` exportados desde el scope contienen los arrays de señal cruda para post-procesamiento en MATLAB. El tamaño de 112KB corresponde a aproximadamente:

```

Trece segundos de señal real del SM-24 en exteriores. Si un auto pasó cerca, ese es el primer evento sísmico registrado con el sistema completo.
